# Архитектура Transformer для русско-английского машинного перевода

Проект содержит реализацию архитектуры Transformer на PyTorch. Модель обучена на датасете OPUS-100 для перевода с русского языка на английский.

Веса модели: https://drive.google.com/drive/folders/1myrFmVeuuM07lNLOnLUPWoY--3BmXane?usp=sharing
---

## Реализованные компоненты

- MultiHeadAttention, PositionalEncoding, Encoder/Decoder Layers, FeedForward, LayerNorm.
- Архитектура соответствует описанию из статьи "Attention Is All You Need".
- Обучен BPE-токенизатор на 32 000 токенов.
- Проведена фильтрация данных: удалены фрагменты с китайскими иероглифами, индийским алфавитом и другими посторонними символами.
- Применено накопление градиентов (Gradient Accumulation) для работы с эффективным размером батча 32 при ограниченном объёме видеопамяти.
- Реализовано автоматическое сохранение чекпоинтов для возобновления обучения.

## Результаты

| Метрика | Значение |
| :--- | :--- |
| BLEU | 20.01 |
| METEOR | 0.3927 |
| TER | 1.0747 |

Полученный показатель BLEU равен 20.01 при эталонном диапазоне от 0 до 100; значения выше 20 считаются минимально достаточными для сохранения общего смысла предложений. Метрика METEOR составила 0.3927 при эталонном интервале от 0 до 1, где диапазон 0.3–0.5 соответствует среднему качеству перевода с передачей ключевой семантики. По метрике TER оптимальным результатом является стремление к 0, а нормой для качественного перевода — диапазон 0.4–0.6. Итоговый показатель TER составил 1.0747. Превышение единицы обусловлено ошибкой генерации — избыточным циклическим повтором токенов.

## Краткое описание работы модели

1. Текст разбивается на подслова с помощью BPE.
2. Кодер обрабатывает входное предложение и формирует его векторное представление.
3. Декодер генерирует перевод пошагово, используя маскированное внимание и внимание к выходу кодера.
4. Выходной слой преобразует результат в вероятности слов целевого языка.
